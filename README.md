# BP-Synth Code Tutorials

STM32-based monosynth project based on the tutorial series by **Blaine Perkins (Synthetech)**.

## Original Author

All original code and tutorials are the work of **Blaine Perkins**.

- YouTube Channel: [Synthetech](https://www.youtube.com/@Synthetech)
- Tutorial Playlist: [BP-Synth Code Tutorials](https://www.youtube.com/playlist?list=PLOjbFVchrTEyJBfXwbhBicpxuOOPgtQ3v)

## About

The BP-Synth is a monosynth built around the STM32F411 (BlackPill) microcontroller, outputting audio
over I2S to a PCM5102 or UDA1337 DAC at 32kHz. The tutorial series builds the synth incrementally —
each project folder adds one feature on top of the last, from bare hardware init through to a fully
featured monosynth with oscillator selection, ADSR envelopes, and a DSP filter.

## Project Directory Structure

Each code project is a self-contained STM32CubeIDE workspace containing:
- `Core/Src/` — application source files (`main.c`, `oscillators.c`, `waveGen.c`, etc.)
- `Core/Inc/` — header files
- `Drivers/` — STM32 HAL and CMSIS drivers (generated by STM32CubeMX)
- `startup/` — startup assembly file
- `*.ioc` — STM32CubeMX hardware configuration file

---

### `BP_Init`
**Hardware initialization only.** Configures the STM32F411 peripherals needed for the synth:
I2S1 in master-transmit mode at 32kHz (for the DAC), USART2 at 31250 baud (MIDI standard), and
DMA2 for streaming audio. The main loop is empty — no audio code yet. This is the starting template
every subsequent project is built from.

---

### `BP_BasicSaw`
**First audio output — sawtooth oscillator.** Introduces the DMA-driven circular audio buffer
(`audiobuff`). The two I2S DMA callbacks (`HAL_I2S_TxHalfCpltCallback` / `HAL_I2S_TxCpltCallback`)
keep the buffer continuously filled by calling `make_sound()`. Calls `Synth_Init()` on startup and
launches the audio stream with `HAL_I2S_Transmit_DMA()`. Produces a continuous sawtooth wave at a
fixed pitch with no MIDI input yet.

---

### `BP_BasicMIDI`
**MIDI note on/off.** Adds interrupt-driven MIDI parsing over UART. A `HAL_UART_RxCpltCallback`
handler assembles 3-byte MIDI messages byte-by-byte and calls `procMIDI()`, which handles note-on
(0x90) and note-off (0x80) commands. A 16-slot voice array tracks held notes, and `op1.amp` is
toggled to gate the oscillator. Oscillator frequency tracks the current MIDI note. Pitchbend (0xE0)
and CC (0xB0) handlers are stubs at this stage.

---

### `BP_Pitchbend`
**MIDI pitchbend.** Activates the 0xE0 pitchbend handler with a working `doPitchBnd()` function.
Decodes the 14-bit pitchbend value, maps it to a semitone range (configurable `bendRange`, default
±6 semitones), and computes a `bendFactor` multiplier using `powf(2, semitones/12)` that is applied
to the oscillator frequency.

---

### `BP_Vibrato`
**LFO vibrato effect.** Builds directly on the Pitchbend project. The vibrato LFO modulates
oscillator pitch periodically, implemented in `oscillators.c` and `parameterFuncs.c`. Rate and depth
are controllable via MIDI CCs. The `main.c` MIDI handling is the same as in Pitchbend.

---

### `BP_MIDI-CCs`
**MIDI Control Change parameter control.** Activates the 0xB0 CC handler by calling
`ProcessRXdMidiCCs()` (defined in `MIDI_CCs.c`). Allows a MIDI controller to adjust synth parameters
(e.g. volume, vibrato rate/depth) in real time. Includes pitchbend support carried over from the
previous project.

---

### `BP_OscSelect_1-2`
**Selectable oscillator waveforms, in two parts.**

- **`BP_OscSelect-1`** — Adds selectable waveforms: sawtooth, square, triangle, and sine. Also
  introduces the ADSR amplitude envelope generator (`ADSR_t amp_EG`). Note on/off now calls
  `ADSR_keyOn()` / `ADSR_keyOff()` instead of toggling `op1.amp` directly, giving the synth proper
  amplitude shaping.

- **`BP_OscSelect-2`** — Extends the oscillator selection further with two additional waveforms:
  pulse (with pulse-width control) and MorphSaw. The ADSR and waveform select infrastructure from
  part 1 is retained.

---

### `BP_ADSR_EG`
**ADSR amplitude envelope generator**, in two variants (`BP_ADSR_EG` and `BP_ADSR_EG_2`). Both
implement the same `ADSR_t amp_EG` envelope that shapes the amplitude of each note through Attack,
Decay, Sustain, and Release stages. `ADSR_keyOn()` is called on note-on and `ADSR_keyOff()` on
note-off. The two variants explore minor differences in the envelope implementation or parameter
initialization.

---

### `BP_Filter_CODE`
**DSP low-pass filter**, in two parts.

- **`BP_Filter-1`** — Introduces a DSP low-pass filter (`DSP-Filter.c`) applied to the audio output.
  Filter cutoff and resonance are controllable via MIDI CCs. Uses the amplitude ADSR from the
  previous project.

- **`BP_Filter-2`** — The most complete version of the synth. Adds a **second ADSR envelope
  dedicated to filter cutoff modulation** (`filterEG`), so the filter sweeps over the course of each
  note. Also adds **legato mode**: when enabled, re-triggered notes do not restart the envelopes,
  producing a smooth, connected playing style. Both the amp EG and filter EG are retriggered on note
  change when legato is off.

---

### `BPsynthFiles_Basic`
**Supporting files for flashing and using the synth hardware.** Contains:
- Pre-built firmware hex files (`BP_Synth_ST-Link.hex`, `BP_Synth_DeFuse.hex.dfu`) ready to flash
- `BP_Synth-GUI.exe` — a Windows GUI for editing synth parameters in real time over MIDI
- `BP-Synth MIDI CC List.pdf` — the full list of MIDI CC assignments
- `BPsynth_Schematic_PCM5102.png` / `BPsynth_Schematic_UDA1337.png` — wiring diagrams for both
  supported DAC modules
- `HowToUseDfuSe.txt` — instructions for flashing via DFU
- Links to ST-Link Utility and DfuSe download pages

## Note on Build Artifacts

Build output directories (`Debug/`, `Release/`) and compiled artifacts (`.o`, `.d`, `.elf`, `.hex`,
`.map`, etc.) have been removed from version control and are excluded via `.gitignore`. These are
generated locally by STM32CubeIDE when you build a project and do not need to be tracked in git.
